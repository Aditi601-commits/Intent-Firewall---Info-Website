2. Vision & Product Intent

The product is currently deployed as IntentFirewall V2, a mobile security application focused on scam and synthetic content defense.

Core goals

  • Protect users from social engineering and scam messages
  • Detect synthetic or deepfake call audio
  • Keep sensitive inference local on device
  • Optionally federate privacy-preserving risk signals

Defense tiers

  • Tier 1: Regex and heuristic filtering
  • Tier 2: Distilled text classifier
  • Tier 3: Additional classifier fusion
  • Audio: TFLite models with LFCC, glottal, and WavLM student features

4. Runtime Architecture Overview

The system spans mobile UI, native Android monitoring, and Python model training.

Mobile app composition

The app boots from index.js and renders App.tsx, which returns the navigation stack for Home, Warning, History, and Settings.

Android text pipeline

Notifications are filtered, batched into context, and scored through Tier 1, Tier 2, and optional Tier 3 detectors before emitting events to React Native.

Android audio pipeline

Call audio is captured in 2-second windows, scored for synthetic speech, and flagged when confidence is high.

Python ML pipeline

Python scripts support dataset building, model training, distillation, export, and evaluation for text and audio detectors.

Federated server

The optional Traverse server manages HMAC-validated submissions, Bloom filters, differential privacy, and reputation scoring.

5. React Native Layer

The React Native app handles UI, settings, threat history, and event bridging from native detectors.

Key screens

  • Home: displays protection state, recent threats, and listens for native notifications.
  • Warning: shows threat details and blocking actions.
  • History: lists past threats with filtering.
  • Settings: user preferences, permissions, and clear history actions.

Threat storage and detection

Threats are saved by ID and include category, confidence, blocked state, and timestamp. Settings persist via AsyncStorage.

The in-app detector uses keyword weighting to score scam content, generate a confidence value, and resolve categories such as OTP, Authority, Urgency, and Impersonation.

6. Android Native Layer

The Android layer is responsible for monitoring notifications, running local classifiers, and bridging events into React Native.

Build setup

Android is configured for SDK 36 with Hermes enabled and includes TFLite plus ONNX runtime dependencies. Services and notification listeners are declared in the manifest.

Notification pipeline

NotificationService captures posted notifications, filters by supported apps, buffers context, and applies tiered detection logic.

Event bridge

NotificationEventEmitter emits native events named onNotification to React Native with classification metadata.

7. Python ML Pipeline

The Python scripts support end-to-end model development, including feature extraction, training, distillation, and export.

  • Dataset building and preprocessing
  • Detector model definitions and training loops
  • Distillation workflows for compact student models
  • ONNX and TFLite export scripts
  • Modern attack evaluation and benchmarking

8. Federated Reputation Server

The optional traverse_server.py provides a privacy-preserving server backend for risk signals.

  • HMAC-validated submissions
  • Epoch-partitioned Bloom filters
  • Differential privacy via Laplace noise
  • Device consensus gating and reputation scoring
